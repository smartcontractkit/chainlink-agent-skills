# API Reference

- [https://docs.chain.link/data-feeds/api-reference](https://docs.chain.link/data-feeds/api-reference) — Reference for reading Chainlink Data Feeds on EVM chains via the proxy using AggregatorV3Interface, and optionally querying the underlying aggregator contract (AccessControlledOffchainAggregator) for additional metadata/config. Explains best practice: call the proxy through AggregatorV3Interface so aggregator upgrades don’t break integrations. Documents core read methods (decimals, description, getRoundData, latestRoundData, version) including return fields (roundId, answer, startedAt, updatedAt, answeredInRound) and notes answeredInRound is deprecated. Covers AccessControlledOffchainAggregator variables (LINK, owner, access controllers, minAnswer/maxAnswer largely unused) and functions for billing/config/transmission details; flags several legacy methods as deprecated (getAnswer/getTimestamp/latestRound/latestTimestamp). Emphasizes checking typeAndVersion() and contract source/config because aggregator variants differ by feed/network.. Key details: Use AggregatorV3Interface pointed at the feed proxy address; example constructor sets priceFeed = AggregatorV3Interface(0x694AA1769357215DE4FAC081bf1f309aDC325306) (Sepolia ETH/USD).; AggregatorV3Interface.decimals() -> uint8; description() -> string; version() -> uint256.; AggregatorV3Interface.getRoundData(uint80 _roundId) and latestRoundData() return (uint80 roundId, int256 answer, uint256 startedAt, uint256 updatedAt, uint80 answeredInRound).; answeredInRound is deprecated (historical multi-round computation field).; AccessControlledOffchainAggregator should be read only when proxy interface lacks needed fields; upgrades behind proxy won’t affect proxy calls.; Use typeAndVersion() on aggregator to identify aggregator variant/version; contract version() is different from typeAndVersion().; AccessControlledOffchainAggregator deprecated methods: getAnswer(), getTimestamp(), latestRound(), latestTimestamp(). Prefer latestRoundData() for freshness timestamps.; AccessControlledOffchainAggregator.getBilling() returns (maximumGasPrice, reasonableGasPrice, microLinkPerEth, linkGweiPerObservation, linkGweiPerTransmission).; minAnswer/maxAnswer are no longer used on most feeds; implement custom circuit breaker if needed (risk mitigation).; To inspect aggregator ABI, fetch from a block explorer (e.g., Etherscan) and use Remix 'AtAddress' pattern.
- [https://docs.chain.link/data-feeds/mvr-feeds/api-reference](https://docs.chain.link/data-feeds/mvr-feeds/api-reference) — API reference for Chainlink Multiple-Variable Response (MVR) Feeds on EVM chains, focusing on reading data via the IBundleAggregatorProxy at the proxy contract address. Explains that the proxy combines IBundleBaseAggregator (bundle-reading functions) and ICommonAggregator (metadata), keeping consumer contracts compatible if the underlying aggregator changes as long as they use the proxy interface. Documents functions to read the latest encoded bundle (bytes), retrieve per-field decimals for numeric fields (non-numeric typically 0), and get the latest update timestamp. Also covers proxy management/view functions for aggregator upgrades (proposedAggregator, confirmAggregator, aggregator) and metadata (description, version). Notes that bundles must be decoded according to the specific feed’s known structure; integration examples are in a separate Solidity guide.. Key details: Primary interface: IBundleAggregatorProxy (used with the proxy contract address); Compatibility note: consumer remains compatible across aggregator changes if interacting through proxy interface; latestBundle() external view returns (bytes memory bundle); bundleDecimals() external view returns (uint8[] memory) where non-numeric fields typically map to 0; latestBundleTimestamp() external view returns (uint256) block timestamp of most recent bundle; proposedAggregator() external view returns (address) proposed replacement aggregator (not active until confirmed); confirmAggregator(address aggregatorAddress) external; typically owner/authorized only; aggregator() external view returns (address) currently confirmed aggregator behind proxy; description() external view returns (string memory); version() external view returns (uint256); Bundle decoding caveat: bytes bundle must be decoded using the known structure for the specific MVR feed
