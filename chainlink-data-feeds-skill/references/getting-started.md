# Getting Started

- [https://docs.chain.link/data-feeds/getting-started](https://docs.chain.link/data-feeds/getting-started) — Explains how to consume Chainlink Data Feeds from an EVM smart contract using Solidity by reading an onchain aggregator (price feed) via AggregatorV3Interface. Walks through a sample DataConsumerV3 contract that hardcodes the Sepolia BTC/USD proxy address, calls latestRoundData(), and returns only the answer. Covers deployment and interaction in Remix with MetaMask on Sepolia, including compiler selection via pragma and the need to use Injected Provider (Web3) rather than a JS VM because the contract calls another onchain contract. Notes key caveats: example is unaudited/not for production; on L2s (Arbitrum/OP/Metis) you must check the L2 Sequencer Uptime Feed; returned answers are integers with feed-specific decimals (e.g., BTC/USD uses 8).. Key details: Solidity pragma: ^0.8.7; Import: @chainlink/contracts/src/v0.8/shared/interfaces/AggregatorV3Interface.sol; Constructor sets dataFeed = AggregatorV3Interface(<feed proxy address>); Function pattern: dataFeed.latestRoundData() returns (roundId, answer, startedAt, updatedAt, answeredInRound); example ignores all but answer; Example feed: Sepolia BTC/USD proxy address 0x1b44F3514812d835EB1BDB0acB33d3fA3351Ee43; Remix: use Injected Provider environment; JavaScript VM will not work for this contract; Decimals are feed-specific; example BTC/USD uses 8 decimals (3030914000000 => 30309.14); L2 requirement: check L2 Sequencer Uptime Feed before trusting price data during sequencer outages; Warnings about unused local variables in Remix can be ignored for this example; Caveat: example uses hardcoded values and unaudited code; do not use in production
