# Price Feeds

- [https://docs.chain.link/data-feeds/price-feeds](https://docs.chain.link/data-feeds/price-feeds) — High-level overview of Chainlink Data Feeds with a focus on Price Feeds. Explains that feeds aggregate data from many sources via a decentralized set of independent node operators (per the Decentralized Data Model), with caveats that some feeds may rely on a single data source or use calculated values. Points readers to guidance on evaluating feed quality (Selecting Quality Data Feeds) and provides navigation to next steps: reading answers from Data Feeds, retrieving historical price data, finding feed contract addresses, and the Data Feeds API Reference. No SDK-specific code patterns or configuration formats are shown on this page.. Key details: Price Feeds are aggregated from many data sources by decentralized independent node operators; Exceptions exist: some feeds may use a single data source or calculated values; Recommends using "Selecting Quality Data Feeds" to identify feed categories and quality; Links to next steps: read answers from Data Feeds, get historical price data, find contract addresses, Data Feeds API Reference
- [https://docs.chain.link/data-feeds/price-feeds/addresses](https://docs.chain.link/data-feeds/price-feeds/addresses) — Directory-style documentation page for Chainlink Data Feeds focused on Price Feed contract addresses across supported networks. It primarily routes builders to the correct address listings and related guides rather than teaching integration code. It emphasizes developer responsibility and due diligence: official documented feeds are reviewed, but community/custom deployments may carry additional risks. It points readers to “Using Data Feeds” for implementation details, “LINK Token Contracts” for LINK token/faucet info, and “Selecting Quality Data Feeds” plus asset-type best practices (ETF/Forex, exchange rates) for data-quality and risk evaluation before integrating feeds into smart contracts.. Key details: Purpose is to provide/point to Price Feed contract addresses (feed addresses) per network; Integration instructions are deferred to the “Using Data Feeds” guide; Data-quality guidance is deferred to “Selecting Quality Data Feeds” and asset-specific best practices (ETF/Forex, exchange rates); Caveat: official documentation feeds are reviewed; community/custom deployments may have additional risks—perform diligence before using in smart contracts; References “LINK Token Contracts” for LINK token and faucet details
