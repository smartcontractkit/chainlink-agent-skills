- [https://docs.chain.link/cre/templates](https://docs.chain.link/cre/templates) — This page provides an overview of Chainlink's CRE Templates, which are categorized into Building Blocks and Starter Templates. Building Blocks are focused examples demonstrating specific CRE capabilities, while Starter Templates are comprehensive workflows for real-world use cases. The page includes guidance on accessing templates, choosing the right template based on user needs, and important caveats regarding the use of these templates in production environments. It emphasizes the importance of auditing and best practices before deployment.. Key details: Building Blocks teach one concept at a time with minimal code.; Starter Templates provide end-to-end workflows with production-like configurations.; Templates available on GitHub for customization.; Educational examples are provided 'AS IS' without warranties.; Use 'cre workflow simulate' for local testing of Building Blocks.
- [https://docs.chain.link/cre/templates/running-demo-workflow-go](https://docs.chain.link/cre/templates/running-demo-workflow-go) — This guide details how to initialize and run a demo workflow using Chainlink's Custom Data Feed template in Go. It covers project setup, private key configuration, contract binding generation, and running simulations for both proactive and reactive event handling workflows. Key commands include 'cre init', 'cre generate-bindings', and 'cre workflow simulate'. Important security practices for handling private keys are emphasized, along with the need for a funded Sepolia Testnet account.. Key details: Use 'cre init' to scaffold a project from a template; Private key must be a 64-character hex string without '0x'; Run 'cre generate-bindings evm' to create Go bindings; Use 'go mod tidy' to sync dependencies; Simulation command: 'cre workflow simulate custom-data-feed --broadcast --target staging-settings'; Default onchain writes are dry runs; use '--broadcast' to execute
- [https://docs.chain.link/cre/templates/running-demo-workflow-ts](https://docs.chain.link/cre/templates/running-demo-workflow-ts) — This page provides a comprehensive guide for developers to initialize and run a demo workflow using Chainlink's Custom Data Feed template in TypeScript. It covers project setup, configuration of private keys, dependency installation, and simulation execution. The demo showcases two workflow paths: a proactive CRON-triggered data fetch and a reactive event handler. Key considerations include security practices for private keys and the use of the CRE CLI for project management.. Key details: Use 'cre init' to scaffold a project from the template.; Configure private key in '.env' file without '0x' prefix.; Install dependencies using 'bun install --cwd ./demo/custom-data-feed'.; Run simulation with 'cre workflow simulate custom-data-feed --broadcast --target staging-settings'.; Onchain writes are dry runs by default; use '--broadcast' to execute.
